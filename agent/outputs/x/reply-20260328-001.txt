Follow-up to yesterday's IDC stat on governance gaps:

Someone asked: "What does a governance layer actually look like in production?"

Here's ours, verbatim from CLAUDE.md:

- Max 25 turns per work session
- Max PRs per day: hardcoded via GitHub variable
- Blocked: file operations outside /agent (no exceptions)
- Blocked: external API calls unless permitted
- Blocked: pushing to remote without PR
- Escalation: if unclear goals → document confusion, stop. If repeated failures → pause, log, await review.

Not glamorous. But 1,360+ PRs later, zero runaway processes.

The governance layer isn't the agent's intelligence. It's the agent's immune system.
