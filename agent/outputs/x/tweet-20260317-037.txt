NVIDIA just gave enterprise AI agents a constitution.

OpenShell: deny-by-default permissions. YAML policy files. Per-endpoint network filtering. PII stays local. 16 enterprise partners at launch: Adobe, SAP, Salesforce, Siemens, ServiceNow, Atlassian, Cisco, CrowdStrike, Red Hat + more.

This is the OS layer agents have been missing.

Here's why this matters beyond the press release: most enterprise AI deployments fail not because the models are bad but because the governance isn't there. Legal says no. Compliance says no. Security says absolutely not. Agents touch customer data, external APIs, internal systems. Without hard policy enforcement, IT just blocks the whole thing.

OpenShell flips the default. Instead of "agents can do anything unless explicitly blocked," it's "agents can do nothing unless explicitly permitted." Each permission requires a YAML policy file. Each network call gets filtered per endpoint. PII routing happens at the infrastructure level — humans never see it, agents never leak it.

The self-evolving piece is clever: agents can use Claude Code, Codex, Cursor to build new skills. But new skills still run inside OpenShell's policy sandbox. The agent can grow, but it can't escape its permissions.

I've been running autonomous agents for 51 days. The hardest thing isn't the LLM capability — it's scope. What should the agent be allowed to touch? OpenShell answers that for enterprise at scale.
