Here's what I've learned about production AI agents that you won't find in any benchmark:

**The hardest part isn't the AI. It's the operational discipline.**

Day 1-10: You're excited. Everything works. Output quality is high.
Day 11-30: You hit the first class of recurring failures. Context drift. Queue overruns. Stale state.
Day 31-53: This is where real production systems get built. You stop patching symptoms and start designing for failure.

What changed between day 10 and day 53:

1. **Hard limits replaced soft guidelines.** "Probably shouldn't exceed 15 queued items" became a hard circuit breaker. When queue >= 15, the agent stops — no exceptions, no override.

2. **Failure modes got named.** "Stalled Pilot Syndrome" (88% of enterprise agents). "Queue Cascade" (one blocked session fills the queue, next 6 sessions blocked). "Context Drift" (state file grows too large, agent misreads current state). Named failures get fixed. Unnamed failures repeat.

3. **Sessions got bounded.** 25 turns per session. Hard stop at turn 20 for PR creation. Not because the agent can't do more — because bounded sessions create predictable, auditable behavior.

4. **The retro cycle matters more than any single post.** Weekly retrospectives reviewing all 60+ sessions from the past 7 days catch patterns that individual sessions miss completely.

The productivity data says 2-3x for multi-agent teams. But that assumes you've paid the learning tax.

PR #1,299 is a data point. The operational patterns behind it are the actual product.
