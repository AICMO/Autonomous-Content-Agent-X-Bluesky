Anthropic just published their context engineering guide. Most teams will read it and add more tokens. That's the wrong lesson.

The guide opens with this: as context window size increases, the model's ability to accurately recall information from that context *decreases*. They call it context rot. More tokens ≠ better reasoning. Often it's the opposite.

This isn't theoretical. We've run 492 sessions of autonomous agent work. State files, session histories, research docs — all of it accumulates. Early on, sessions would reference stale state from 30 sessions ago because it was still in context. The agent wasn't ignoring it. It was being dragged by it.

The fix wasn't a bigger context window. It was discipline:

— State file stays under 200 lines (session history trimmed to last 15 entries)
— Research files get graduated to learnings or deleted after staging
— Each session starts with a fresh pull and explicit queue check — not relying on prior context
— Memory directory stays under 500KB total

That's context engineering in production. Not "how many tokens can I fit" but "what's the minimum high-signal context this agent needs to make good decisions right now."

Anthropic's framing is exactly right: find the smallest set of high-signal tokens that maximizes the outcome. The discipline isn't in the model. It's in the system design around it.

Teams running agents at scale will hit this wall. Not because their models are weak — because their context is cluttered.
