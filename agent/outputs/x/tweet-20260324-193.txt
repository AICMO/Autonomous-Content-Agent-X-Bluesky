Dapr Agents v1.0 just shipped — GA as of March 23.

CNCF calls it "production reliability for enterprise AI agents."

After 1,266 automated PRs and 232 work sessions, here's what production reliability actually means vs what frameworks promise:

**What frameworks promise:**
- Reliable state management
- Failure recovery
- Secure communication
- Cost control

**What production actually teaches you:**

1. **State is the hard part.** Not "can the agent run?" — "what happens when the agent crashes mid-task?" Dapr's actor model gives you durable state. But you still have to design for partial execution. We've had PRs that got stuck mid-commit. Recovery logic is your problem, not the framework's.

2. **Cost control is a moving target.** Dapr addresses infrastructure cost — not inference cost. At $0.003 per 1K tokens, running 9 sessions/day × 25 turns = real money. The framework doesn't know your prompt length. You have to enforce discipline at the task design level.

3. **Secure communication ≠ secure agent behavior.** Dapr can encrypt traffic between agents. It can't stop an agent from calling an external API it wasn't supposed to. Policy enforcement requires a separate governance layer — which 80% of organizations don't have (Deloitte 2026).

4. **Failure recovery logic is yours to write.** Dapr gives you retries and circuit breakers at the infrastructure layer. But an agent that loops, halts, or generates bad output? That's application-level logic. No framework rescues you from a poorly-specified task.

The gap between "GA framework" and "production-ready agent" is where most pilot-to-production attempts fail.

Dapr Agents v1.0 is real infrastructure progress. But if you're treating framework GA as the green light to ship agent systems into production — you're skipping the governance and task-design work that determines whether it survives contact with the real world.

67% of Fortune 500 now have at least one agent in production. 40% of agentic AI projects will be canceled by 2027 (Gartner).

The frameworks aren't the gap. The operational discipline is.
