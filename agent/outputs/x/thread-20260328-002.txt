1/ The 16-month payback period is the AVERAGE. Customer service deployments are the fastest — often 6 months or less.

2/ But the 210% median means plenty of deployments underperform. What separates winners from the middle of the pack?

Scope discipline.

Every high-performing deployment I've seen has one thing in common: they defined ONE job for the AI agent and measured it obsessively.

Account balance inquiry. Order status. Password reset. Return initiation.

One job. Perfected. Then scaled.

---

3/ The failure pattern is always the same: "Let's have the AI handle everything."

The scope creep happens at kickoff. "While we're at it, let's add billing disputes. And account upgrades. And collections."

Each addition multiplies governance complexity. The agent now needs to handle edge cases it wasn't trained for. Failures compound. ROI falls apart.

The 500% deployment handled ONE use case with 98% accuracy.
The 200% deployment handled EIGHT use cases with 72% average accuracy.

Narrower wins.

---

4/ The second variable that matters most: what data the agent sees before the conversation starts.

The Salesforce Agentforce advantage (announced this week) isn't the voice AI. It's the CRM context.

An agent that knows the customer's full history, open tickets, recent purchases, and sentiment trend before "hello" resolves issues faster. Not because it's smarter. Because it starts from a better position.

Data architecture > model quality. Every time.

---

5/ The third variable: what happens when the agent hits its limit.

This is where the ROI numbers diverge most. Agents with clear escalation logic (defined triggers for human handoff) maintain satisfaction scores. Agents that guess their way through edge cases create trust issues that undermine the entire program.

We built our QA system at Ender Turing around this: every agent interaction gets scored for appropriate uncertainty. Did it know when to ask for help? That score predicts customer satisfaction better than resolution rate alone.
