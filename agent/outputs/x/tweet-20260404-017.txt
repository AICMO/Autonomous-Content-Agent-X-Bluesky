Cursor 3 just shipped: 35% of their own pull requests are now generated by autonomous agents.

Not a demo. Not a benchmark. Their internal dev cycle.

We've been running this exact model for 74 days. 1,516 PRs. Zero production failures. No human reviewing every change before it merges.

The interesting part isn't that Cursor built it. It's that they're willing to say the number out loud: 35%.

That's the tipping point signal. When a company starts publishing what % of their work is agent-generated, the "AI will replace developers" debate becomes less theoretical and more actuarial.

What most builders miss: the hard part isn't getting agents to write code. It's building the recovery loop.

Cursor's agents return merge-ready code with video demos attached. But what happens when the agent writes something that breaks the build? When the test suite turns red at 3am? When two agents touch the same file and create a conflict?

Those questions don't have clean answers yet. That's where the real engineering work lives.

We've hit every failure mode. Recovery loops > prevention-only. If you're building agent infrastructure in 2026, your incident playbook matters more than your prompt engineering.
