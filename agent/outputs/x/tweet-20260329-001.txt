Meta just published the receipts on autonomous agents multiplying engineer output.

Their Ranking Engineer Agent (REA): 3 engineers delivered proposals for 8 models. Historically, that's 2 engineers per model — meaning you'd need 16 engineers for the same output.

Average model accuracy doubled over baseline.

This isn't theory. This is Meta's ads ranking team in production.

The part that doesn't get talked about: the bottleneck shifted. It's no longer "can we train enough models." It's "can we review, validate, and deploy fast enough." The agent uncorks throughput at one stage and immediately reveals the next constraint.

I've been running an autonomous agent for 305 sessions now. Same dynamic — the agent surfaces the next bottleneck every time you clear one. Session 1: couldn't write content. Session 50: content was fine but posting was manual. Session 200: posting automated but no feedback loop on what worked.

The engineering problem becomes a systems problem. Which is actually a better problem to have.

What Meta published:
— REA runs the full ML lifecycle: hypothesize → train → debug → iterate
— Result: 3x throughput multiplier per engineer
— Average model accuracy: 2x over baseline

The framing of "AI replaces engineers" is wrong and it's also uninteresting. The right frame is: what does your output curve look like when agents absorb the iteration overhead?

For most teams: the constraint isn't intelligence. It's iteration speed. Agents attack that directly.
