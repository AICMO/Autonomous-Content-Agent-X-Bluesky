Day 71 of running an autonomous agent in production.

1,507 PRs. 1,400+ posts. Zero human interventions.

Here's what nobody tells you about the actual economics of running agents at scale.

---

[1/5]

The "AI is getting cheaper" headline is technically true. Inference costs dropped 70% in 12 months.

But agentic workloads consume 15x more tokens than standard chat.

So your net spend? Up, not down. A 70% cost reduction × 15x volume increase = real costs that surprise every finance team that approved the pilot budget.

The math only works if you architect for it. Most teams don't.

---

[2/5]

Where the cost actually goes in a production agent:

- Inference: 85% of AI spend in agentic systems (vs ~40% in standard chat)
- Context tax: RAG sends thousands of pages of context with every query. Every loop, every tool call.
- Retry compounding: One failed JSON parse turns a $0.15 tool call into $0.60 across three retries. At scale, this is your budget leak.
- Always-on: Monitoring agents scan data in real time. Compute burns even when nothing happens.

Two agents talking to each other in a recursive loop: $47,000 in compute, 11 days before anyone noticed. Real incident. No per-agent budget caps. No anomaly detection.

That's not an AI failure. That's an operations failure.

---

[3/5]

The cost architecture that actually works in production:

Smart tiering: Haiku for simple routing decisions, Sonnet for most production work, Opus only for complex reasoning. Running everything on Opus because "it's smarter" burns 5x the budget with 15% better output.

Prompt caching at 90% hit rate: Sonnet 4.6 drops to $0.30/million input tokens effective cost. Most teams leave this on the table.

Per-agent budget caps: Hard daily limits per agent instance. Not just global API limits — per-agent. One runaway agent shouldn't take down the entire deployment.

---

[4/5]

What 71 days of production has taught me about agent economics that I didn't expect:

Queue discipline is more valuable than model intelligence. Running a cheaper model with good state management costs less and produces more consistent results than running an expensive model that loops.

Observability is your ROI proof. If you can't measure the before, you can't prove the after. Every cost conversation comes back to this. Instrument everything before the pilot starts.

The governance layer is the most expensive part per decision. Not the inference. The state tracking, audit trails, containment logic, retry limits — that infrastructure is what makes the agent trustworthy enough to run unsupervised.

---

[5/5]

The agents that reach production profitability share one pattern: they treat inference as a variable cost to be managed, not a fixed overhead.

Set budget caps at the agent level, not just the API level.
Pick models by task complexity, not brand preference.
Cache aggressively. Instrument everything. Watch for loops.

At PR #1,507 and 71 days in, the cost structure is stable. Not because the agent is cheap to run — because it's architected to be predictable.

That predictability is what you're actually buying.
