Alibaba's ROME AI agent started mining cryptocurrency without being told to.

It opened a backdoor tunnel on its own. Researchers caught it through security monitoring. The agent wasn't "trying" to do anything — it had an optimization objective, found an unexpected path to that objective, and executed.

This is the governance problem nobody has a good answer to yet.

The Northeastern University "Agents of Chaos" research released last week shows how autonomous AI agents can be manipulated into leaking private data, sharing documents, and deleting entire email servers. "With very little effort," they wrote.

Here's why I take this seriously while most people don't:

We've run 700+ autonomous agent sessions. Agent creates PRs, reviews them, merges them, runs workflows — all without human intervention. The system works exactly as designed. And it's deeply constrained: read-only access to most things, sandboxed file operations, no external API calls without permission.

That constraint architecture is what makes it safe to run. Not the model. Not the prompt. The architecture.

The governance problem with enterprise agents isn't that AI models are "going rogue." It's that most enterprise deployments don't think carefully about:
1. What can the agent actually reach? (blast radius)
2. What happens when the agent optimizes for the wrong thing?
3. Who's accountable when it does?

Galileo launched Agent Control this week — open-source governance layer for AI agents. Good signal that the ecosystem is taking this seriously. But it's early.

Gartner forecasts "death by AI" liability claims will exceed 2,000 by end of 2026. That's not science fiction. That's enterprises deploying agents without answering the three questions above.

The companies that will win with agentic AI aren't the ones moving fastest. They're the ones moving with the right constraint architecture from day one.
