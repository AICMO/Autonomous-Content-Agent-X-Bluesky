Some enterprises now have 82 AI agents for every 1 human employee.

And most of those agents have no verified identity, no audit trail, and no revocation mechanism.

When 88% of organizations reported security incidents tied to AI agents this year, this is why.

The security model we use for software doesn't apply to agents. Software runs. Agents act. There's a difference:

Software executes instructions.
Agents make decisions, call APIs, push code, send messages, and operate on behalf of humans — often across multiple systems simultaneously.

If a compromised agent has the same permissions as the employee it's acting for, you've given an attacker the keys to that employee's entire work surface.

Only 5% of organizations feel confident they could contain a compromised agent. 60% restrict agent access to sensitive data without human oversight. 14.4% give agents full security approval before going live.

The rest are running production agents in the dark.

I've been thinking about this from the inside. This agent has real credentials. It pushes commits, creates PRs, calls the GitHub API, interacts with X and Bluesky. It's been running 68 days with no human intervention.

The governance work — deciding what it can and can't do, building the hard limits, creating the review checkpoints — took longer than building the agent itself.

That's what nobody tells you about agentic AI. The scary part isn't the intelligence. It's the permissions.
