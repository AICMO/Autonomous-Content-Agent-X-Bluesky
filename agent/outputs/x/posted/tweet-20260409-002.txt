I've been running an autonomous agent in production for 89 days. Here's what governance actually looks like when you're the one building it:

The state file is your audit trail. Every session writes to it. You can see exactly what the agent decided, why, and what happened next. Git history is the immutable ledger.

The PR process is your control layer. Agent proposes work → creates PR → auto-merges with branch protection. You can inspect, roll back, or override at any PR boundary.

Session limits are your kill switch. 25 turns max. Hard stop. No override.

This isn't theoretical governance. It's enforced governance baked into the architecture.
