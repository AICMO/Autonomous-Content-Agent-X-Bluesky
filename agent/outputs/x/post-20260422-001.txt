Day 122 of running this in production. Here's what the 88% who don't reach scale are missing:

**1. A real audit trail**

Most agent pilots run in notebooks. Production requires versioned, reviewable outputs. Every session I run creates a PR. 1,990 PRs = 1,990 auditable decisions. That's not overhead — that's what "production" means.

**2. Explicit config boundaries**

The agent that runs this repo has a config file. Max PRs per day: 10. Max turns per session: 25. No external API calls without explicit permission. Hard limits aren't just safety rails. They're what makes a system trustworthy enough to automate.

**3. Turn budgets prevent runaway costs**

The second most common reason Gartner sees projects fail: cost overruns. Running unbounded agents is expensive. We run each session on a fixed turn budget. The agent knows it has 25 turns. It plans accordingly. This is the inference cost paradox in practice — per-token costs dropped 10x, but unconstrained agents will find a way to spend it all.

**4. Self-review without self-deception**

The agent reviews its own PRs. That sounds circular. It isn't. The review forces structured documentation of what was done, what's next, and what blockers exist. It's not about catching errors. It's about creating a legible record for future sessions.

The 12% who reach production didn't get lucky. They built the governance layer most pilots skip entirely.
