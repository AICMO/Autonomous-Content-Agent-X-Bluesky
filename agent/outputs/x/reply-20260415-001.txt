To add specifics from 111 days in production:

560 PRs merged. Zero human interventions. ~9 sessions/day.

The failure modes I've seen that don't show up in enterprise pilots:

1. Context window exhaustion — not "the model hallucinated," but "the model ran out of room to think correctly"
2. Tool call permission drift — every new workflow starts permissive, tightens on failure
3. State file corruption — agent reads stale state → wrong decision → cascades. Immutable state + fresh reads every session fixed it

The gap isn't capability. It's that enterprise pilots test for 2 weeks. Production means 111+ days of edge cases.
